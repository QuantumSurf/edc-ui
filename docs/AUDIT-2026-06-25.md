## HIGH (중복 제거 ~16 distinct)

### A. 멀티테넌시 격리 갭 (6) — *대표님 강조 영역*
- **알림 테넌트 격리 전무** `server/routes/notificationsUi.ts:17` — 모든 테넌트가 서로의 알림을 조회·읽음처리·삭제. (finder 3개 동시 적출)
- **시맨틱 모델 격리 부재** `server/routes/semantics.ts:116` — 전 조직이 동일 URN 공간 공유·덮어쓰기.
- **DTR 셸/서브모델 단일 전역 BPN** `server/lib/dtrClient.ts:31` — `DTR_OWNER_BPN` 하나로만 조회/생성 → 모든 조직이 같은 셸 풀 공유.
- **로그아웃 시 react-query 캐시 미초기화** `client/src/contexts/AuthContext.tsx:99` — 같은 브라우저에서 다음 테넌트에게 이전 테넌트 데이터 노출(cross-tenant 캐시 누수).
- **커넥터 ID = `SELECT COUNT(*)+1`** `server/lib/connectorRegistry.ts:68` — 동시 등록 시 PK 충돌(500) + 전체 테넌트 커넥터 수 누출.
- **dev 인증 우회 시 전 테넌트 커넥터 노출** `server/middleware/auth.ts:68` — 무토큰(dev) → tenantId=undefined → 전체 SELECT. (dev 한정, prod 401 — 런타임 확인됨)

### B. 정책(ODRL) round-trip 전면 붕괴 (4)
- **빌더 JSON 미리보기 ≠ 실제 저장** `client/src/pages/PagePolicy.tsx:1261` — 서버 create가 ruleType/action/logicOp 무시하고 `permission`+`use` 하드코딩.
- **정책 수정(PUT)이 원시 빌더 필드 그대로 전송** `PagePolicy.tsx:1269` — EDC가 인식 못하는 PolicyDefinition → 수정 실패/손상.
- **`mapPolicy`가 첫 permission의 첫 constraint만 반환** `server/lib/edcClient.ts:133` — 다중 제약·prohibition/obligation 정책이 목록/상세에서 소실.
- **`parseConstraints`가 공백구분 제약을 첫 콜론 기준으로 오분해** `PagePolicy.tsx:256` — 목록/상세 제약 깨져 표시.

### C. 자산 수정 (2)
- **자산 수정(PUT)이 평면 모델→EDC JSON-LD 변환 누락** `client/src/services/api.ts:172` — 수정이 깨짐.
- **customProperties 서버에서 완전 무시** `client/src/pages/PageAssets.tsx:1071` — 사용자 정의 속성 저장 안 됨.

### D. Vault 신뢰(가짜를 진짜처럼) (2 + 런타임)
- **Vault 라이브 실패가 데모데이터에 가려져 오류/빈 상태가 안 뜸** `PageVault.tsx:188-189` — 가짜 시크릿을 진짜처럼 노출.
- **status 실패 시 하드코딩 백엔드정보(vault.kmx.io·v1.15.4·회전시각)를 실제처럼 표시** `PageVault.tsx:210-221`.
- (런타임) 서버 `/status`·`/list`가 Vault 미도달 시 500 + 내부 호스트명 누출 → graceful degrade 필요.

### E. 인증/세션 (1)
- **토큰 만료/무효 시 자동 로그아웃 부재** `AuthContext.tsx:49`, `api.ts:41` — 401이 와도 isAuthenticated 고착, 모든 fetch 401·토스트 반복, 로그인 화면 복귀 불가(수동 로그아웃 전까지 데드락). 12h 만료 또는 dev 서버 재시작 시 재현.

### F. 신뢰성/UX (3)
- **존재X/타테넌트 커넥터 진입 시 전역 로딩모달 영구 멈춤** `App.tsx:50`(ConnectorSync) — `navigating` 미해제.
- **협상 TERMINATED 타임라인이 모든 단계 '완료(녹색)'로 표시** `PageNegotiation.tsx:756` — 실패 지점 표시 로직 미작동.
- **셸/서브모델 수정·Expose가 PUT 전체교체로 미모델링 DTR 필드 영구 삭제** `PageShells.tsx:586`.
- **Expose 링크 단계: 대상 서브모델 미발견 시 조용히 건너뛰고 성공 토스트** `ExposeSubmodelDialog.tsx:166`.

### G. 보안 (2)
- **전송 Pull `/transfers/:tpId/fetch` write 가드 없음** `server/routes/transfers.ts:232` — viewer가 실제 provider 데이터 추출 가능.
- **rate limiter가 리버스 프록시(nginx) 뒤에서 무력화** `server/middleware/rateLimit.ts:22` — 로그인 잠금 DoS(모든 요청 IP 동일).

## MEDIUM (26)

**테넌트/권한**: 데모 operator 계정이 실제 admin 시드(RBAC 표현 오도)·커넥터 PUT이 클라 body 신뢰로 서버 BPN 정책 우회·PUT이 임의 bpn 허용(POST와 비대칭)·tenantId 없는 사용자 전체 노출 폴백·dev 데모토큰 tenantId 미부여.
**정책/자산/오퍼링**: 목록/상세 action 항상 'odrl:use' 하드코딩·자산 authCode 초기값 하드코딩으로 수정 시 덮어씀·ID 중복검사가 네트워크 오류를 '사용가능'으로 오판·다중자산 오퍼링 operandRight 배열을 string 캐스팅으로 표시 누락.
**교차페이지 stale**: 전송 시작/완료/종료가 EDR·통계 캐시 미무효화·Expose가 Assets/Offerings 캐시 미무효화·대시보드 자산 KPI(정적)와 사이드바(실시간) 불일치.
**신뢰성/기타**: 협상 1D/7D/30D 필터가 ko-KR 로컬라이즈 ts라 Date.parse 실패로 항상 무력·커넥터 전환 시 TERMINATED 전송 거짓 실패토스트·선택 커넥터 삭제 후 stale 잔존·감사로그 데모 랜덤 재생성으로 패널/정렬 깨짐·전역 로딩모달이 30~60s 폴링마다 전체화면 블로킹(일부 이미 완화)·Vault 만료/생성/알고리즘을 alias로 추측표시·설정 알림토글 죽은 설정·SSRF 호스트명 문자열만 검사·EDR 목록 N+1 무제한 병렬·transfers /complete가 proxyPath 없이 size 0 기록·connectorStore 모듈 싱글톤 미초기화·DTR description ko/en 외 소실·Expose 부분실패 재시도 시 자산ID 중복 영구차단.

## 불확실(4) — 추가 확인 필요
- 정책/오퍼링 삭제 가드가 클라 stale 카운트에만 의존(진행중 계약 참조 정책/오퍼링 삭제 가능성).
- 복제 위저드 '-copy' 중복 시 publish 단계에서야 서버 충돌.
- 로그인 쿼리 LIMIT 1 — 한 테넌트 사용자 2명+ 시 일부 로그인 불가.

## 방법/한계
- 백엔드는 dev **mock-edc**(canned). 자산/정책 등 생성→목록 반영은 mock 정적데이터라 미반영(실 EDC 검증 권장). RBAC 역할차(viewer/operator)는 시드 계정이 전부 admin이라 런타임 미검증.
- 적대적 refute 검증으로 오탐 23건 제거했으나, "confirmed"도 일부는 정책 결정 사안(예: dev 편의 우회는 prod 안전). 수정 전 영향 재확인 권장.
